# 学习优先级建议

> 生成日期：2026-04-29  
> 资料定位：AI Agent 应用岗位的系统学习 + 工程实战参考资料。  
> 来源基础：用户指定 GitHub 目录、公开 JD 调研、官方框架文档与经典论文。  
> 注意：招聘信息会变化，岗位调研用于能力画像，不代表某公司当前仍在招聘。

## 1. 总体策略

你的目标是 Python 技术栈下的 AI Agent 应用岗位，因此学习优先级应是：

```text
先能做出可控 Agent
再能接入知识和工具
再能上线、评测、监控和治理
最后补微调和框架横向选型
```

不要一开始陷入模型训练细节，也不要只学框架 API。框架 API 会变，但 Workflow、状态、工具、上下文、评测、安全这些能力更稳定。

## 2. 四阶段学习计划

### 阶段 1：建立 Agent 系统大图

精读：01、02、03。  
目标：能解释 Agent 与 Chatbot、Chain、Workflow 的区别；能画出 Agent 的规划、执行、观察、状态更新闭环；能说明工具调用为什么需要后端治理。

### 阶段 2：掌握 RAG 与上下文

精读：04、05、06。  
目标：能设计企业知识库 RAG，能解释 Chunking、Embedding、Hybrid Search、Rerank、Lost in the Middle、记忆写入/读取策略。

### 阶段 3：掌握 Python 工程化与生产治理

精读：08、09、10。  
目标：能把 Agent 做成服务，讲清 FastAPI、asyncio、队列、Redis、PostgreSQL、Trace、监控、HITL、审计、沙箱。

### 阶段 4：项目、框架和面试表达

精读：11、12、14、15、16。  
目标：能用 LangGraph 讲一个端到端项目，能回答框架选型和生产化问题，能把简历项目讲成指标闭环。

## 3. 7 天学习建议

- Day 1：01 Agent 基础 + 02 Workflow。
- Day 2：03 工具调用 / MCP + 14 LangGraph 基础映射。
- Day 3：04 RAG 全链路。
- Day 4：05 上下文与记忆 + 06 优化策略。
- Day 5：08 Python 后端 + 16 Lab 代码骨架。
- Day 6：09 评测监控 + 10 安全风控。
- Day 7：11 项目设计 + 12 面试题 + 13 Checklist。

## 4. 学习产出要求

每学完一章，至少产出三样东西：

```text
1. 一张文字版架构图；
2. 一套面试回答；
3. 一个项目落地点或代码骨架。
```

如果只停留在概念解释，很难和 JD 里的“生产级 Agent”要求匹配。
