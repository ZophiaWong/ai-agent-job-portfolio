## 1. 评分方法

综合考虑 5 个维度：

```text
JD 必备项出现频率：35%
工程落地价值：25%
面试考察概率：20%
生产风险相关性：10%
与你的 Python 技术栈相关性：10%
```

分数不是数学精确统计，而是基于公开 JD 样本、官方框架趋势和 Agent 应用工程实践形成的优先级判断。

## 3. P0 能力解释

### 3.1 RAG 与知识库工程

P0 的原因是企业大模型应用大多绕不开私有知识：客服、内部知识库、代码助手、数据分析、合规问答、销售助手都需要把模型和外部知识连接起来。RAG 的关键不是“向量库 + prompt”，而是从数据接入到答案引用的完整链路。

必须掌握：文档解析、清洗、Chunking、Embedding、向量检索、BM25、Hybrid Search、Rerank、Query Rewrite、Citation、RAG Eval、索引版本化、Bad Case 回流。

### 3.2 Agent Workflow / LangGraph 状态编排

Agent 任务一旦超过单轮问答，就需要可控编排。LangGraph 的 State、Node、Edge、Conditional Edge、Checkpoint、Interrupt 等概念，正好对应工程里的状态、步骤、分支、恢复和人工审批。

必须掌握：Workflow 与 Agent 的区别、Planner-Executor、Router、Evaluator-Optimizer、多 Agent 协作、终止条件、最大步数、Checkpoint、HITL。

### 3.3 Tool Calling / MCP

Agent 的差异化能力来自“能行动”。但行动必须受控，否则会引入越权、重复副作用、工具失败、数据泄露等风险。

必须掌握：工具 schema、结构化输出、服务端参数校验、权限校验、幂等键、可重试/不可重试错误、MCP 工具发现与调用、审计日志。

### 3.4 Python 后端工程能力

应用岗位需要把 Agent 做成服务，而不是 notebook demo。FastAPI、asyncio、任务队列、缓存、数据库、容器、日志、监控、部署和降级是面试和项目表达的重要抓手。

必须掌握：API 层、异步并发、任务队列、Redis、PostgreSQL、pgvector、Vector DB、OpenTelemetry、Docker、限流、熔断、状态一致性。

## 4. P1 能力解释

P1 能力通常不是每条 JD 都写，但在中高级岗位、系统设计、项目深挖中非常重要，包括上下文工程、评测监控、安全风控、框架选型和项目表达。它们决定你的回答是否像“真正做过生产系统”。

## 5. P2 能力解释

微调、LoRA、QLoRA、DPO、RLHF 是重要基础，但在 Agent 应用岗中通常属于加分项。它们适合在以下场景深入：

- 岗位明确要求模型适配或训练；
- 你需要解释为什么不用微调；
- 项目中 Prompt/RAG/Workflow 已达到瓶颈；
- 你想证明自己理解模型层优化边界。
