## 1. 本章学习目标

本章不是训练大模型的完整课程，而是从 AI Agent 应用工程角度理解模型调优和微调。

你需要掌握：

```text
预训练、SFT、RLHF、DPO 的定位；
LoRA、QLoRA 为什么能低成本微调；
微调适合解决什么问题；
微调不适合解决什么问题；
Agent 轨迹数据如何构造；
微调模型如何评测、灰度和回滚。
```

对于应用岗位，重点不是推导算法公式，而是知道何时需要微调、如何准备数据、如何证明微调有效，以及如何避免把 RAG/工具/Workflow 问题误判成模型问题。

## 2. 核心概念总览

### 2.1 LLM 训练与后训练链路

```text
预训练：
  大规模文本上学习语言和世界知识。

SFT：
  有监督指令微调，让模型学会遵循指令和对话格式。

偏好优化：
  通过人类或 AI 偏好，让模型输出更符合人类期望。
  常见方法包括 RLHF、DPO、RLAIF。

参数高效微调：
  不更新全部参数，而是训练少量适配参数。
  常见方法包括 LoRA、QLoRA。
```

### 2.2 BERT 与 GPT 类模型的区别

```text
BERT：
  Encoder-only，适合理解、分类、匹配、抽取。

GPT 类 LLM：
  Decoder-only，自回归生成，适合对话、生成、推理、工具调用、Agent。
```

Agent 应用主要使用生成式 LLM，但在某些 RAG 场景中，BERT 类模型或 cross-encoder 仍可用于 rerank、分类、意图识别。

## 3. 关键知识点详解
### 3.1 SFT

**是什么：** SFT 是用高质量指令-回答样本对模型进行有监督训练，让模型学会特定任务、格式和表达方式。

**为什么需要：** 基础模型会预测下一个 token，但不一定会遵循人类指令。SFT 可以让模型更稳定地执行任务格式。

**核心机制：** 训练数据通常包含 instruction、input、output，也可以包含多轮对话、工具调用轨迹和领域样本。

**工程实现：** Agent 场景中，SFT 可用于稳定工具调用格式、特定业务问答风格、标准报告结构、代码解释模式等。

**常见坑：**

- 数据质量低。
- 样本覆盖不足。
- 把错误答案训练进去。
- 过拟合格式导致泛化变差。

**优化方式：**

- 清洗训练数据。
- 保留验证集。
- 加入反例和边界样本。
- 上线前做回归评测。

**面试表达：** SFT 适合让模型学会稳定执行某类任务，但它不是补充实时知识的首选，实时知识应优先 RAG。
### 3.2 RLHF / DPO / RLAIF

**是什么：** RLHF、DPO、RLAIF 都是偏好优化方法，用来让模型输出更符合人类或 AI 评价偏好。

**为什么需要：** 有些问题没有唯一标准答案，需要根据偏好判断哪个回答更好，例如更有帮助、更安全、更简洁。

**核心机制：** RLHF 通常包含奖励模型和强化学习优化；DPO 使用偏好对直接优化策略；RLAIF 用 AI 反馈替代部分人工反馈。

**工程实现：** 应用工程中不一定亲自训练 RLHF，但需要理解偏好数据、评价标准和对齐对模型行为的影响。

**常见坑：**

- 偏好数据不一致。
- 奖励 hacking。
- 模型变得过度拒答或过度迎合。
- 评测只看主观评分。

**优化方式：**

- 明确偏好 rubric。
- 多人标注和冲突仲裁。
- 加入安全和事实性评测。
- 保持对通用能力的回归测试。

**面试表达：** DPO 相比复杂 RLHF 流程更轻量，常用于偏好对齐；但应用岗更常见的是使用已对齐模型并设计好评测和 guardrails。
### 3.3 LoRA 与 QLoRA

**是什么：** LoRA 是冻结基座模型，只训练低秩增量矩阵的参数高效微调方法；QLoRA 在量化基座模型基础上进一步降低显存成本。

**为什么需要：** 全量微调成本高、显存要求大。LoRA/QLoRA 让小团队也能针对特定行为或领域表达做适配。

**核心机制：** LoRA 通过低秩矩阵近似权重更新；QLoRA 通常将基座模型量化到低 bit，并训练 LoRA adapter。

**工程实现：** 工程上可为不同业务任务训练 adapter，通过模型服务加载或切换。需要关注延迟、兼容性和回滚。

**常见坑：**

- 以为 LoRA 能注入大量实时知识。
- 训练数据过少但期望过高。
- adapter 管理混乱。
- 量化导致精度下降未评测。

**优化方式：**

- 先做小规模验证。
- 明确 adapter 适用任务。
- 建立 adapter registry。
- 对比基座模型、RAG 方案和微调方案。

**面试表达：** LoRA/QLoRA 更适合行为、格式和领域表达适配，不适合替代知识库。
### 3.4 Agent 轨迹数据

**是什么：** Agent 轨迹数据记录模型在任务中的计划、工具调用、观察结果、修正和最终答案。

**为什么需要：** 如果要让模型学会更稳定的工具使用和任务执行，仅训练最终答案是不够的，需要训练过程轨迹。

**核心机制：** 轨迹可以包括 task、state、available_tools、action、arguments、observation、next_decision、final_answer 等字段。

**工程实现：** 数据来源包括专家示范、线上成功案例、失败案例修正、合成任务和沙箱环境。必须过滤错误轨迹，避免训练模型学坏。

**常见坑：**

- 直接收集线上日志训练，含大量失败和噪声。
- 暴露敏感数据。
- 轨迹缺少状态和工具描述。
- 没有标注为什么某步是正确的。

**优化方式：**

- 先做脱敏。
- 人工审核关键样本。
- 失败轨迹转成修正轨迹。
- 构建任务成功率评测集。

**面试表达：** Agent 微调不只是训练最终回答，而是训练可执行决策轨迹。高质量轨迹比数量更重要。
### 3.5 微调上线工程

**是什么：** 微调上线工程是把训练后的模型或 adapter 安全地接入生产系统，并证明它比基线更好。

**为什么需要：** 微调可能改善某些能力，也可能破坏其他能力。没有评测和灰度，上线风险很高。

**核心机制：** 流程包括目标定义、数据构建、训练、离线评测、安全评测、回归评测、影子流量、小流量灰度、监控和回滚。

**工程实现：** LLMOps 中需要记录数据版本、模型版本、adapter 版本、评测结果、上线时间和回滚方案。

**常见坑：**

- 没有明确目标。
- 只看训练 loss。
- 没有和 Prompt/RAG 方案比较。
- 上线后无法回滚。

**优化方式：**

- 定义业务指标。
- 建立 holdout 测试集。
- 做 A/B 或 shadow test。
- 保留基座模型和旧 adapter 回滚路径。

**面试表达：** 微调上线不是训练结束，而是评测、灰度、监控和回滚的完整工程流程。

## 5. 工程实战设计

### 5.1 Agent 微调数据格式示例

```text
{
  "task": "根据用户问题检索知识库并回答",
  "available_tools": ["search_kb", "rerank", "citation_check"],
  "state": {
    "user_query": "...",
    "retrieved_docs": []
  },
  "trajectory": [
    {
      "action": "search_kb",
      "arguments": {"query": "...", "top_k": 10},
      "observation": {"doc_ids": ["d1", "d2"]}
    },
    {
      "action": "rerank",
      "arguments": {"doc_ids": ["d1", "d2"]},
      "observation": {"top_docs": ["d2"]}
    }
  ],
  "final_answer": "...",
  "quality_label": "success"
}
```

### 5.2 微调前的检查清单

```text
是否有明确指标？
是否已有 Prompt/RAG/工具链路优化基线？
训练数据是否脱敏？
是否有失败案例修正样本？
是否有 holdout 测试集？
是否有安全评测？
是否有回滚方案？
```

## 7. 常见误区

### 7.1 把微调当成知识库

微调不适合存放频繁变化的事实和私有数据。知识更新应优先 RAG，微调适合行为模式和表达方式。

### 7.2 没有数据就谈微调

微调成败高度依赖数据。没有高质量样本和评测集时，微调往往只是增加成本和风险。

### 7.3 只看模型分数，不看系统指标

Agent 是系统，微调后还要看任务成功率、工具调用准确率、成本、延迟、安全拒答和用户反馈。

## 8. 面试题与答题闭环
### Q：SFT、RLHF、DPO 的区别是什么？

**考察点：** 后训练基本概念。

**推荐回答：** SFT 是用标准答案教模型遵循指令和任务格式；RLHF 是通过奖励模型和强化学习优化人类偏好；DPO 则直接用偏好对优化模型，流程比 RLHF 更简单。应用岗重点是知道它们分别解决指令遵循和偏好对齐问题。

### Q：LoRA 和 QLoRA 的区别是什么？

**考察点：** 参数高效微调、应用边界。

**推荐回答：** LoRA 冻结基座模型，只训练低秩增量参数，降低微调成本；QLoRA 在量化基座模型的基础上训练 LoRA adapter，进一步降低显存需求。它们适合行为和领域表达适配，但不适合替代 RAG 存储实时知识。

### Q：Agent 微调需要什么数据？

**考察点：** 轨迹数据、工具调用训练。

**推荐回答：** 不只需要最终答案，还需要任务、可用工具、状态、动作、参数、观察结果、修正过程和最终结果等轨迹数据。数据应来自专家示范、成功案例和修正后的失败案例，并经过脱敏、过滤和评测。
