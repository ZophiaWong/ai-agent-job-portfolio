# JD 差异层与实用入口

这里不再按岗位名复制学习路线。RAG 工程师、Agent 应用工程师、Python AI 后端和 LLMOps 的共享能力都引用[C01–C13 核心能力模型](../../.codex/skills/interview-prep-coach/references/competency-model.md)。

## 收到一份 JD 后

1. 把 JD 的职责和追问映射到已有 C ID。例如，RAG 检索设计映射 C05，RAG 评测映射 C06，LangGraph 状态编排映射 C03。
2. 建立 JD 覆盖，只修改优先级和目标等级。不给同一能力换 ID，不因为 JD 提到某个框架就改写历史证据。
3. 在[手册索引](../00_README_学习路线与资料使用说明.md)查缺口，然后走[2–3 周证据路线](../Part_04_项目与面试表达/13_总复习Checklist与学习计划.md)。
