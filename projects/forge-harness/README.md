# Forge Harness Interview Evidence

Source: [ZophiaWong/forge-harness](https://github.com/ZophiaWong/forge-harness)

Forge Harness is a TypeScript tutorial project that grows a real LLM/tool loop into a governed
coding Agent harness through small runnable checkpoints.

## Competency Anchors

- C03: runtime state, tasks, background work, child sessions, and handoff.
- C04: tool registry, permission decisions, reviewable edits, and action boundaries.
- C06: trace evidence, verification, recovery, and observable lifecycle events.
- C08: permission governance and worktree isolation.
- C11: TypeScript/Node runtime, context projection, compaction, and async sessions.
- C10: explaining why each mechanism exists and what remains outside the tutorial scope.

Before a project mock, verify the latest completed checkpoint in the public README. Keep target
architecture and future chapters separate from implemented behavior.
