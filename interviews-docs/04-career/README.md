# 简历、自我介绍与行为面试

本目录提供通用方法。真实简历、自我介绍、项目贡献和行为故事应保存在
`.local/interview-prep/`，不得提交到公开仓库。

## 简历

每条技术主张应能回答四个问题：

1. 解决了什么真实问题？
2. 你本人做了什么决定或实现？
3. 为什么采用这个方案，有哪些取舍？
4. 有什么可检查的代码、测试、trace、指标或演示证据？

优先写完整闭环和个人贡献，避免只罗列框架名。MeterDesk 与 Forge Harness 的公开事实边界
见[项目证据索引](../../projects/README.md)。

## 自我介绍

60 秒版本建议结构：目标岗位 → 可迁移背景 → 两个项目的最强证据 → 当前匹配点。

2 分钟版本可增加一次关键技术决策和结果，但所有主张都应能承受项目追问。分别维护两个
版本，通过计时模拟验证，不靠继续加长解决表达不清。

## 行为面试

使用具体事实组织故事：背景与约束、你的行动、结果证据、复盘。至少准备：

- 主动承担和推进；
- 技术分歧与沟通；
- 失败、排障和恢复；
- 在不确定领域快速学习；
- 跨角色协作。

Judge 应评价行动是否具体、结果是否有证据、反思是否真实；不能因为故事语言流畅就推断
能力。故事中涉及公司、个人或面试隐私时，只记录在本地私人状态。

## Codex 练习入口

```text
Use $interview-prep-coach to stress-test the claims in my private resume.
用 60 秒自我介绍模式测试我，只在结束后反馈。
基于我的真实经历进行一次行为面试，不要替我编故事。
```
