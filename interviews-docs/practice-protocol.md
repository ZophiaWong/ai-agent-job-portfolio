# 统一练习协议

所有专题共用下面的练习顺序：

```text
冷启动 → 澄清约束 → 独立作答/编码 → 执行或检查 → 解释权衡 → 迁移题 → 延迟复测
```

1. **冷启动**：不看笔记，用 1–3 分钟说出定义、适用场景和一个例子，暴露真实起点。
2. **澄清约束**：确认输入输出、规模、正确性、延迟、成本、安全和是否允许修改数据。
3. **独立作答/编码**：先给出自己的方案；需要代码时写出可运行的最小实现。
4. **执行或检查**：运行测试、手动走边界 case，或用可检查的清单核对设计与项目主张。
5. **解释权衡**：说明为何选择该方案、替代方案何时更好，以及失败模式和缓解方式。
6. **迁移题**：改变一个关键约束（规模、失败模式、数据形态或安全级别），重新选择方案。
7. **延迟复测**：隔天或更晚重做冷启动；只把仍不稳的知识点记录为下一轮练习目标。

练习记录应保存可检查的结论、测试结果或证据链接；真实简历、项目贡献和面试反馈只保存在
`.local/interview-prep/`，不提交到公开仓库。
