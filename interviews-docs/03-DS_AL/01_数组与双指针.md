# 01_数组与双指针

本章覆盖数组、字符串、哈希、前缀和、排序与区间、双指针。面试中这些题通常不难在算法名，而难在能否快速识别“哪些状态可以被一次扫描维护”。

## 面试识别信号

- 题目要求查找、计数、去重、分组、是否出现过：优先考虑 `dict` / `set`。
- 题目涉及连续子数组和、区间统计、出现次数：考虑前缀和或前缀计数。
- 题目数组已排序，或可以排序后处理：考虑左右指针、同向指针、区间合并。
- 题目要求原地修改数组：注意写指针和读指针分离。
- 题目出现 interval、meeting、merge、overlap：通常先排序再扫描。

## 核心套路

- 哈希表把“查找是否存在”从 `O(n)` 降到均摊 `O(1)`。
- 双指针适合已排序数组、两端收缩、原地去重、快慢写入。
- 前缀和把区间和 `sum(i..j)` 转成 `prefix[j + 1] - prefix[i]`。
- 区间题先按起点排序，再维护当前合并区间的右边界。
- 面试中先说暴力解，再指出重复计算或重复查找如何被数据结构消掉。

## Python 模板

```python
from collections import defaultdict


def count_items(nums):
    counter = defaultdict(int)
    for x in nums:
        counter[x] += 1
    return counter
```

```python
def two_pointers_sorted(nums, target):
    left, right = 0, len(nums) - 1
    while left < right:
        total = nums[left] + nums[right]
        if total == target:
            return [left, right]
        if total < target:
            left += 1
        else:
            right -= 1
    return []
```

```python
def prefix_sum(nums):
    prefix = [0]
    for x in nums:
        prefix.append(prefix[-1] + x)
    return prefix
```

## 讲解例题

### LeetCode 1. Two Sum

思路：遍历时用哈希表保存“值 -> 下标”。当前数 `x` 需要的另一个数是 `target - x`，如果已经出现过，就直接返回。这样避免每个元素再向后找一遍。

```python
from typing import List


class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        seen = {}
        for i, x in enumerate(nums):
            need = target - x
            if need in seen:
                return [seen[need], i]
            seen[x] = i
        return []
```

复杂度：时间 `O(n)`，空间 `O(n)`。

### LeetCode 56. Merge Intervals

思路：先按起点排序。维护已经合并出的最后一个区间，如果新区间起点不超过最后区间终点，就更新终点；否则新开一个区间。

```python
from typing import List


class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        if not intervals:
            return []

        intervals.sort(key=lambda item: item[0])
        merged = [intervals[0]]

        for start, end in intervals[1:]:
            last = merged[-1]
            if start <= last[1]:
                last[1] = max(last[1], end)
            else:
                merged.append([start, end])

        return merged
```

复杂度：排序 `O(n log n)`，扫描 `O(n)`，空间取决于输出。

## 常见坑

- `dict` 里保存的是值、下标还是计数，要在写代码前说清楚。
- 前缀和计数题要先放入 `counter[0] = 1`，否则漏掉从 0 开始的子数组。
- 原地去重题不要边遍历边 `remove`，容易退化到 `O(n^2)`。
- 区间合并要先排序；只比较相邻原始区间通常是错的。
- 双指针只有在有单调性时才可靠，比如排序数组的两数和。
- `merge` 中的 `intervals.sort(...)` 会原地重排输入；调用方需要保留顺序时先传入副本。

## 面试表达

可以这样讲：暴力做法需要重复查找或重复计算，我用哈希表、前缀和或排序后的双指针把重复工作压掉。核心不在 API，而在维护一个能快速回答问题的状态。
