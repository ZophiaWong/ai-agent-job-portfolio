# 03_链表

链表题重点不是 API，而是指针更新顺序。面试中要主动说明空链表、单节点、偶数/奇数长度、是否会断链。

## 面试识别信号

- 题目要求原地修改节点连接关系，而不是只读值。
- 出现 reverse、cycle、middle、merge、remove nth、reorder。
- 只能顺序访问，不能通过下标随机访问。
- 需要 `O(1)` 额外空间时，通常要直接改 `next` 指针。

## 核心套路

- 反转链表：保存 `next_node`，再改 `cur.next`，最后整体前进。
- 快慢指针：判断环、找中点、找倒数第 N 个节点。
- dummy 节点：统一处理头节点可能变化的情况。
- 合并链表：用尾指针逐步接入较小节点。
- LRU 思路：哈希表定位节点，双向链表维护最近使用顺序。

## Python 模板

```python
from typing import Optional


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


def reverse(head: Optional[ListNode]) -> Optional[ListNode]:
    prev = None
    cur = head
    while cur:
        next_node = cur.next
        cur.next = prev
        prev = cur
        cur = next_node
    return prev
```

```python
def has_cycle(head: Optional[ListNode]) -> bool:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False
```

## 讲解例题

### LeetCode 206. Reverse Linked List

思路：用 `prev` 表示已经反转好的头部，用 `cur` 遍历原链表。每次先保存 `cur.next`，否则改指针后会丢失剩余链表。

```python
from typing import Optional


class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        prev = None
        cur = head

        while cur:
            next_node = cur.next
            cur.next = prev
            prev = cur
            cur = next_node

        return prev
```

复杂度：时间 `O(n)`，空间 `O(1)`。

### LeetCode 141. Linked List Cycle

思路：快慢指针从头出发。若存在环，快指针最终会追上慢指针；若不存在环，快指针会先到空。

```python
from typing import Optional


class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        slow = fast = head

        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow is fast:
                return True

        return False
```

复杂度：时间 `O(n)`，空间 `O(1)`。

## 常见坑

- 改 `cur.next` 前必须保存后继节点。
- 头节点可能被删除或替换时，用 dummy 节点。
- 判断环要用节点身份比较，不能只比较节点值。
- 快慢指针循环条件通常是 `while fast and fast.next`。
- 合并链表后别忘了接上剩余部分。
- `reverse` 和多数重连题会原地修改节点的 `next`；若调用方还要原链表，必须先复制节点而不是只复制头指针。
