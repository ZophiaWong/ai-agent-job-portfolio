## Principles

1. 澄清输入、输出和约束
2. 先给出朴素解法
3. 识别可优化的数据结构
4. 写出可运行代码
5. 主动覆盖空输入、重复元素、极值和越界
6. 给出时间和空间复杂度
7. 解释为什么选择该方案

## 面试答题流程

1. 先复述题意，确认输入规模、返回值、是否允许修改原数据。
2. 给出朴素解法，并指出瓶颈是重复查找、重复计算还是搜索空间过大。
3. 识别可维护的状态：哈希表、窗口、指针、栈、队列、堆、递归返回值或 DP 状态。
4. 写出可运行的 Python 代码，优先用标准库：`dict`、`set`、`deque`、`heapq`。
5. 用 2-3 个边界 case 手动跑：空输入、单元素、重复元素、极值或无解情况。
6. 明确时间复杂度和空间复杂度，并说明为什么每个元素被处理几次。
