# 05_动态规划

动态规划适合“当前答案由更小子问题推出”的题。面试中不要先背公式，先说状态定义、转移来源、初始化和遍历顺序。

## 面试识别信号

- 题目问最值、方案数、是否可达，并且有重复子问题。
- 选择当前元素会影响后续状态，例如偷或不偷、选或不选。
- 路径题只能向右或向下，天然有阶段顺序。
- 背包题出现容量、目标和、能否凑出、最少硬币。
- 序列题出现子序列、最长、编辑、匹配。

## 核心套路

- 状态定义：`dp[i]` 或 `dp[i][j]` 表示什么，必须能一句话说清。
- 转移方程：当前状态从哪些更小状态来。
- 初始化：空输入、起点、容量为 0、第一行第一列。
- 遍历顺序：保证计算当前状态时依赖状态已经存在。
- 空间优化：只有在一维滚动不破坏依赖方向时再做。

## Python 模板

### House Robber（一维 DP）

前提：房屋金额为非负数且不能偷相邻两间。定义 `dp[i]` 表示考虑第 `0..i` 间房屋时，能偷到的最高金额；第 `i` 间房屋要么不偷、沿用 `dp[i - 1]`，要么偷它并接上 `dp[i - 2]`。

```python
def rob(nums):
    if not nums:
        return 0
    if len(nums) == 1:
        return nums[0]

    dp = [0] * len(nums)
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])
    for i in range(2, len(nums)):
        dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])
    return dp[-1]
```

转移：`dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])`，循环从 `i = 2` 开始；保留数组以便面试时直观看到状态与转移。

```python
def zero_one_knapsack(weights, values, capacity):
    dp = [0] * (capacity + 1)
    for weight, value in zip(weights, values):
        for c in range(capacity, weight - 1, -1):
            dp[c] = max(dp[c], dp[c - weight] + value)
    return dp[capacity]
```

## 讲解例题

### LeetCode 70. Climbing Stairs

思路：到第 `i` 阶只有两种来源：从 `i - 1` 走一步，或从 `i - 2` 走两步。因此 `dp[i] = dp[i - 1] + dp[i - 2]`。

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 2:
            return n

        prev2, prev1 = 1, 2
        for _ in range(3, n + 1):
            cur = prev1 + prev2
            prev2, prev1 = prev1, cur

        return prev1
```

复杂度：时间 `O(n)`，空间 `O(1)`。

### LeetCode 322. Coin Change

思路：`dp[x]` 表示凑出金额 `x` 的最少硬币数。对每个金额尝试最后一枚硬币，如果 `x - coin` 可达，就更新 `dp[x]`。

```python
from typing import List


class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        inf = amount + 1
        dp = [inf] * (amount + 1)
        dp[0] = 0

        for x in range(1, amount + 1):
            for coin in coins:
                if x >= coin:
                    dp[x] = min(dp[x], dp[x - coin] + 1)

        return -1 if dp[amount] == inf else dp[amount]
```

复杂度：时间 `O(amount * len(coins))`，空间 `O(amount)`。

## 常见坑

- 没有清楚定义 `dp[i]`，代码通常会乱。
- 初始化不能靠猜，要从状态定义推出。
- 0/1 背包一维优化要倒序遍历容量；完全背包通常正序。
- 序列 DP 的答案不一定是 `dp[-1]`，例如 LIS 可能要取全局最大。
- 遇到负数、任意路径、环状依赖时，不一定能直接套普通 DP。

## 面试表达

可以这样讲：我先定义 `dp` 的含义，再说明当前状态依赖哪些已经计算过的状态。这样可以证明遍历顺序和复杂度，而不是直接背公式。
